# IMPLEMENTATION — מה נבנה ומה נשאר למלא

מסמך זה מסכם את מה שמומש באתר "פסטיבל נוסטלגיה", את ההחלטות שהתקבלו, ובעיקר את
**מה שעדיין צריך להשלים לפני העלייה לאוויר**.

---

## סטאק טכנולוגי
- **Frontend:** React 18 + TypeScript, מאוגד עם **Vite**.
- **עיצוב:** CSS ידני בסגנון Windows XP "Luna" (ללא ספריית UI), RTL מלא, עברית.
- **Backend / תיעוד:** **Supabase** (Postgres + REST), מהדפדפן דרך `@supabase/supabase-js`.
- **פריסה:** **GitHub Pages** עם GitHub Actions (build + deploy).
- אין שרת משלנו ואין מפתחות סוד בקוד — הכל סטטי. מפתח ה-Publishable של Supabase
  (`sb_publishable_…`) ציבורי והטבלה מוגנת ע"י RLS (ראו `SUPABASE_SETUP.md`).

## מה מומש (לפי הדרישות)
- **תחושה נוסטלגית / עיצוב XP-2000:** טפט "Bliss" (משוחזר ב-CSS, ללא תמונה מוגנת),
  חלונות בסגנון Luna עם כפתורי כותרת, שורת משימות עם כפתור "התחל" ירוק ושעון חי,
  פונט Tahoma, אייקוני שולחן עבודה. הכל בעברית RTL.
- **פרטי הכנס:** יום שני 29.6.2026, פתיחת דלתות 17:30, במה ראשית 20:00, מיקום
  "אקספו תל אביב, ביתן 11A" — בחלון `details.txt`. מקור הנתונים: `src/lib/config.ts`.
- **ספירה לאחור** חיה לתאריך/שעת הפתיחה (כפי שמופיע במוקאפ ההירו).
- **מידע בלי לחשוף הכל — "קיר הגורלות":** החלון האמצעי עוצב מחדש בהשראת קיר
  הגורלות של ששטוס. יש 5 מפתחות, לוחצים על כדורים והם מתהפכים. כל הפיכה חושפת את
  הפריט הבא ברצף (ששטוס → מסיבה → סופרגול → פוגים → קורנפלקס) — לא משנה על איזה
  כדור לוחצים. כשנגמרים המפתחות מופיעה ההודעה "מהדורות חדשות יוצגו בעתיד".
- **רכישת כרטיסים:** אשף 3 שלבים שאוסף **שם מלא, גיל, מספר כרטיסים, ועם מי מגיעים**
  (חברים/משפחה/ילדים/לבד), **מחשב מחיר** לפי הקלט, ובלחיצה על "מעבר לתשלום"
  **שומר את הפרטים ב-Supabase ואז מפנה** לשירות התשלום. התיעוד קורה *לפני* ההפניה,
  כי ייתכן שלא נוכל להחזיר את הגולש אחרי התשלום (כפי שצוין בדרישות).
- **פריסה:** מוכן ל-GitHub Pages עם Actions; Supabase מתועד במלואו.

## החלטות עיצוב/מימוש שכדאי שתדעו
- **`base: './'`** ב-Vite — האתר עובד גם ב-`user.github.io` וגם ב-
  `user.github.io/repo` בלי לדעת מראש את שם הריפו.
- **תיעוד לא חוסם תשלום:** אם השמירה ל-Supabase נכשלת (רשת/הגדרה), הגולש עדיין
  מופנה לתשלום. לא "מאבדים" לקוח בגלל תקלת לוג.
- **בלי מפתחות בקוד:** כל הקונפיג מגיע ממשתני סביבה בזמן build. בלי הגדרה, האתר
  עדיין נבנה ורץ — התיעוד פשוט הופך ל-no-op (מודפס ל-Console), וכפתור התשלום מציג
  הודעת "מצב הדגמה".
- **פרטיות:** ב-RLS אין מדיניות קריאה ל-anon, כך שאף גולש לא יכול לשלוף רשומות של
  אחרים. רק בעל הפרויקט רואה את הנתונים בדשבורד.

## ⚠️ מה עדיין צריך למלא / להחליט (TODO)

1. **קישור התשלום (`VITE_PAYMENT_URL`)** — חובה. קבלו מההנהלה את כתובת הסליקה
   והגדירו כמשתנה בריפו (ראו `SUPABASE_SETUP.md` חלק ג'.2). בלעדיו כפתור התשלום
   מציג "מצב הדגמה".

2. **מפתחות Supabase** (`VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`) —
   חובה לתיעוד. ראו `SUPABASE_SETUP.md`.

3. **מחירי הכרטיסים — PLACEHOLDER!** הלוגיקה ב-`src/lib/pricing.ts` היא המצאה
   סבירה ולא מאושרת:
   - מחיר בסיס לכרטיס: **₪149**.
   - הנחות לפי מלווים: חברים 5%, משפחה 10%, ילדים 15%.
   - הנחות גיל: עד גיל 12 → 50%, מגיל 65 → 20% (לא מצטבר — נלקחת ההנחה הגבוהה).
   **החליפו במחירון האמיתי של ההנהלה** (הקבועים מרוכזים בראש הקובץ).

4. **תוכן הפעילויות** — כרגע placeholders בלבד ברשימת `REVEALS` שב-
   `FatesWall.tsx` (ששטוס, מסיבה, סופרגול, פוגים, קורנפלקס). מספר המפתחות `K`
   נגזר אוטומטית מאורך הרשימה. כשיהיה תוכן סופי, עדכנו את `REVEALS`.

5. **טקסטים שיווקיים** — הכותרות והתיאורים ב-`WelcomeWindow.tsx` ו-
   `WhatsThereWindow.tsx` נכתבו לפי המוקאפים; אפשר לחדד לפי הקמפיין.

6. **דומיין מותאם אישית** — להגדרה מאוחר יותר ב-Settings → Pages (כפי שציינתם).

7. **(רשות) קישורי אייקוני שולחן העבודה / תפריט "התחל"** — כרגע דקורטיביים. כפתור
   "התחל" וה-CTA גוללים לאשף הכרטיסים.

8. **(רשות) אנליטיקס / פיקסל פרסום** — אם רוצים לייחס המרות לקמפיינים בסושיאל,
   להוסיף פיקסל (Meta/TikTok) ל-`index.html`.

## הרצה מקומית
```bash
npm install
cp .env.example .env      # מלאו ערכים אם רוצים לבדוק Supabase/תשלום מקומית

npm run build             # build לפרודקשן לתיקיית dist/
npm run preview           # תצוגה מקומית של ה-build
```
